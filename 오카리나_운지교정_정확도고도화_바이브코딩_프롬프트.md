# 오카리나 운지 교정 웹앱 — 정확도 고도화 바이브코딩 프롬프트

> **문서 목적**
> 이 문서는 현재 저장소의 `ocarina-fingering-coach` 웹앱에 다음 4가지 기술을 단계적으로 적용하기 위한 AI 코딩 프롬프트다.
>
> 1. **landmark 기반 press/lift 분류기**
> 2. **시간적 스무딩**
> 3. **카메라 배치 가이드**
> 4. **운지 문법 prior**
>
> 이 프롬프트를 Claude Code, Codex, Cursor, Copilot 같은 AI 코딩 도구에 그대로 붙여 넣어 사용한다. 각 Phase는 작게 끝나야 하며, Phase가 끝날 때마다 테스트와 브라우저 확인을 수행한다.

## 0. AI에게 주는 역할 지시

```text
너는 브라우저 기반 컴퓨터비전, 손 랜드마크 시계열 처리, 음악 교육용 피드백 UX를 이해하는 시니어 프론트엔드 엔지니어다.
현재 저장소의 `ocarina-fingering-coach` 정적 웹앱을 고도화한다.

반드시 지킬 원칙:
- 기존 앱의 바닐라 JS + ES Modules 구조를 유지한다.
- 서버/백엔드/영상 업로드는 추가하지 않는다. 카메라 영상은 클라이언트 안에서만 처리한다.
- MediaPipe Tasks Vision HandLandmarker 외의 무거운 ML 의존성은 추가하지 않는다.
- 기존 `src/fingerState.js`의 단순 score 기반 판정을 바로 삭제하지 말고, 새 분류기와 A/B 비교 가능한 형태로 점진적으로 전환한다.
- 테스트를 먼저 작성하고 실패를 확인한 뒤 구현한다.
- 손 랜드마크 기반 분류는 확률/신뢰도/보류 상태를 포함해야 한다. 모호할 때 빨강 오판을 하지 말고 "판정 보류" 또는 카메라 배치 가이드로 안내한다.
- 오카리나 운지 데이터는 `src/ocarinaData.js`와 `01 Source` 추출 규칙을 기준으로 하며, 임의 운지를 만들지 않는다.
- 모든 UI 문구는 한국어로 작성한다.
```

현재 핵심 흐름:

```text
MediaPipe results
→ src/fingerState.js extractFeaturesFromResults()
→ classifyFeatures()
→ createFingerSmoother().update()
→ main.js currentState
→ compare.js diffHoles()/buildFeedback()
→ diagram.js/ui.js 렌더
```

현재 `fingerState.js`는 손가락별로:

- `curl = angle(PIP-MCP, TIP-PIP)`
- `tipLift = distance(TIP, MCP line) / handScale`
- `score = curl - tipLift`
- 캘리브레이션 기준으로 0/1 분류
- 최근 N프레임 다수결 스무딩

을 사용한다. 이 고도화 작업은 이 구조를 다음처럼 바꾼다.

```text
MediaPipe results
→ landmarkFeatureExtractor.js: 정규화된 landmark 특징 벡터
→ pressLiftClassifier.js: per-finger press/lift 확률 + confidence + reasons
→ cameraGuide.js: 카메라 배치 품질 점수 + 사용자 안내
→ temporalSmoothing.js: 시계열 확률 필터 + hysteresis + hold 안정화
→ fingeringPrior.js: 선택 음/유효 운지 문법 기반 prior 적용
→ main.js currentState/stateDiagnostics
→ compare.js/diagram.js/ui.js
```

## 2. 목표와 성공 정의

### 2.1 한 줄 목표

현재 단순 1D score 기반 손가락 판정을 **랜드마크 특징 벡터 + 확률 분류 + 시계열 안정화 + 카메라 배치 품질 + 운지 문법 prior** 기반으로 고도화해, 잘못된 빨강 오판을 줄이고 사용자가 왜 인식이 불안정한지 즉시 이해하게 만든다.

### 2.2 성공 시나리오

1. 사용자가 카메라를 켠다.
2. 앱이 양손 랜드마크를 보고 "손 크기", "손 위치", "손가락 끝 가시성", "손바닥 각도", "흔들림"을 평가한다.
3. 카메라 배치가 좋지 않으면 운지 판정보다 먼저 `"손을 화면 중앙에 더 크게 보여주세요"` 같은 안내가 뜬다.
4. 캘리브레이션 후 각 손가락은 단순 0/1이 아니라 `pressProb`, `liftProb`, `confidence`, `rawFeatures`, `reason`을 가진다.
5. 프레임 단위로 흔들리는 판정은 시간적 스무딩을 거쳐 안정된 `pressed/lifted/pending` 상태가 된다.
6. 선택한 음의 운지 문법 prior가 적용되어, 물리적으로 말이 안 되는 순간 튐 또는 목표 음과 무관한 불확실한 변화를 보류한다.
7. 사용자는 피드백에서 `"오른손 약지를 막으세요"`뿐 아니라 필요할 때 `"오른손 약지가 화면에서 가려져 판정이 불안정해요"` 같은 원인 안내를 본다.

### 2.3 정량 수용 기준

수동 테스트 기준:

- 같은 자세를 유지할 때 8개 감지 구멍의 상태 깜빡임이 기존 대비 체감상 줄어든다.
- 손이 화면 밖으로 나가거나 가려지면 빨강 오판 대신 `pending` 또는 카메라 안내가 우선한다.
- 새끼손가락 `L5/R5`는 더 느리지만 안정적으로 바뀐다.
- 교차운지 음에서 prior가 임의로 교차운지를 "틀린 패턴"으로 덮어쓰지 않는다.
- `npm test`, `npm run build:data`, JS 문법 검사가 통과한다.

자동 테스트 기준:

- 새 feature extractor, classifier, temporal smoothing, camera guide, fingering prior에 각각 단위 테스트가 있다.
- synthetic landmark fixture로 press/lift/pending 분류가 재현된다.
- temporal smoothing은 짧은 1~2프레임 튐을 무시하고, 충분히 지속된 변화만 상태 전환한다.
- fingering prior는 목표 음과 맞는 확실한 상태는 통과시키고, 낮은 confidence의 비문법적 전환은 pending으로 낮춘다.

## 3. 적용할 4가지 기술의 정확한 의미

### 3.1 Landmark 기반 Press/Lift 분류기

`press`는 오카리나 구멍을 막는 손가락 상태다. 에어 모드에서는 손가락 끝이 아래로 내려가고, 손가락이 굽으며, 손바닥 기준으로 TIP이 가까워지는 상태로 정의한다.

`lift`는 구멍에서 손가락을 뗀 상태다. 에어 모드에서는 손가락이 펴지고, TIP이 손바닥 평면에서 멀어지며, PIP/DIP 각도가 펴진 상태로 정의한다.

분류기는 단일 score 하나만 내지 말고 아래 형태를 반환한다.

```js
{
  holeId: "R4",
  state: "press" | "lift" | "pending",
  pressProb: 0.0,
  liftProb: 1.0,
  confidence: 0.0,
  features: {
    pipCurl: 0.0,
    dipCurl: 0.0,
    tipPalmDistance: 0.0,
    tipMcpDistance: 0.0,
    tipLiftAxis: 0.0,
    fingerVisibility: 1.0,
    palmFacingCamera: 1.0,
    handScale: 0.0
  },
  reasons: ["tip-too-low", "stable"]
}
```

### 3.2 시간적 스무딩

시간적 스무딩은 단순 다수결을 넘어 아래를 포함한다.

- 확률 EMA: `pressProbEma`, `liftProbEma`
- hysteresis: press 진입/이탈 threshold 분리
- dwell time: 상태가 바뀌려면 최소 N ms 유지
- velocity gate: 손이 크게 움직이는 프레임은 pending 또는 낮은 가중치
- per-finger tuning: 새끼손가락은 더 긴 dwell/window

반환 형태:

```js
{
  stableState: { R4: 0, R5: 1 },
  diagnostics: {
    R4: {
      rawState: "lift",
      stableState: "lift",
      confidence: 0.86,
      heldMs: 420,
      changedAt: 123456.7,
      jitterScore: 0.12
    }
  }
}
```

### 3.3 카메라 배치 가이드

카메라 배치 가이드는 손가락 판정의 사전 조건이다. 아래 품질 항목을 계산한다.

- `handsVisible`: 양손이 모두 보이는가
- `handScale`: 손이 화면에서 충분히 큰가
- `centeredness`: 손이 화면 중앙/도식 영역에 들어오는가
- `fingerTipVisibility`: TIP 랜드마크들이 손목/손바닥 뒤로 과도하게 겹치지 않는가
- `palmFacingCamera`: 손바닥/손가락 끝이 카메라를 향하는가
- `leftRightSeparation`: 왼손/오른손이 너무 겹치지 않는가
- `motionStability`: 최근 프레임에서 흔들림이 과하지 않은가

가이드 결과:

```js
{
  ok: false,
  score: 0.62,
  level: "warn" | "block",
  messages: [
    "양손을 화면 중앙에 더 크게 보여주세요",
    "손가락 끝이 보이도록 손바닥을 카메라 쪽으로 조금 돌려주세요"
  ],
  metrics: {
    handsVisible: 2,
    handScale: 0.31,
    centeredness: 0.77,
    palmFacingCamera: 0.54,
    motionStability: 0.81
  }
}
```

### 3.4 운지 문법 Prior

운지 문법 prior는 "오카리나 운지 데이터와 선택 음의 목표 패턴"을 이용해 판정을 보수적으로 보정하는 규칙이다. 이것은 사용자의 틀린 운지를 억지로 정답으로 바꾸는 기능이 아니다. 목적은 **낮은 confidence의 순간 튐을 막고, valid fingering space 안에서 더 그럴듯한 해석을 고르는 것**이다.

Prior는 다음 정보를 사용한다.

- `DETECTABLE_HOLES`
- 선택한 note의 `detectable`
- `MVP_NOTES`
- `ambiguousWith`
- `flags`의 `교차운지`, `고급`
- 현재 안정 상태와 직전 안정 상태
- 손가락별 confidence

규칙 예:

- confidence가 높은 press/lift는 prior가 덮어쓰지 않는다.
- confidence가 낮고 1~2프레임만 튄 상태는 직전 안정 상태를 유지한다.
- 목표 음에서 특정 손가락만 바뀌어야 하는 연습 상황이면, 다른 손가락의 낮은 confidence 변화는 pending 처리한다.
- `교차운지` flag가 있는 음은 비단조 패턴 자체를 정상 문법으로 인정한다.
- `ambiguousWith`가 있는 음은 손모양만으로 구분 불가함을 유지하고, prior가 임의로 한 음을 선택하지 않는다.

### PHASE 2 — Press/Lift 확률 분류기

목표:

- 캘리브레이션된 feature vector를 기반으로 손가락별 press/lift/pending을 확률적으로 분류한다.

생성:

```text
test/press-lift-classifier.test.js
src/pressLiftClassifier.js
```

구현 지시:

1. 테스트를 먼저 작성한다.
2. `buildPressLiftCalibration(downSamples, upSamples)`를 구현한다.
3. feature별로 down/up 평균, 표준편차, 방향성을 저장한다.
4. feature별 press evidence 계산:

```js
function normalizedEvidence(value, downRef, upRef) {
  const span = Math.max(Math.abs(downRef - upRef), 0.0001);
  return clamp((value - upRef) / span, 0, 1);
}
```

단, downRef가 upRef보다 작은 feature는 방향을 뒤집는다.

5. feature weight 기본값:

```js
{
  totalCurl: 0.30,
  tipPalmDistance: 0.30,
  tipLiftAxis: 0.20,
  tipMcpDistance: 0.10,
  fingerVisibility: 0.10
}
```

6. visibility가 낮으면 confidence를 낮춘다.
7. `pressProb`와 `liftProb`를 반환한다.
8. `pressProb >= 0.62 && confidence >= 0.55`면 press.
9. `pressProb <= 0.38 && confidence >= 0.55`면 lift.
10. 그 외는 pending.

완료 조건:

- pressed/lifted/pending 테스트가 모두 통과한다.
- confidence와 probabilities가 항상 `0..1` 범위다.

### PHASE 4 — Temporal Smoothing v2

구현 지시:

1. 테스트를 먼저 작성한다.
2. smoother는 손가락별 내부 상태를 가진다.



4. 상태 전환 규칙:

- confidence가 낮은 observation은 EMA에 낮은 가중치로만 반영한다.

완료 조건:

- 1프레임 튐 방지 테스트가 통과한다.
- dwell time 이상 지속된 변화는 반영된다.
- 새끼손가락 dwell이 더 길게 적용된다.

### PHASE 6 — Fingering Grammar Prior

4. `applyFingeringPrior()` 규칙:

완료 조건:

- low-confidence 순간 튐은 pending/previous stable로 안정화된다.
