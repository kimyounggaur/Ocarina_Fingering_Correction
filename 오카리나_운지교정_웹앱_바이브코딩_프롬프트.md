## 1. 프로젝트 개요 & 성공 정의

### 1.1 한 줄 요약
사용자가 **오카리나 종류 + 계이름**을 선택 → 웹캠으로 손 운지를 인식 → 표준 운지와 **손가락(구멍) 단위로 비교** → "어떤 손가락을 어느 구멍에서 막거나 떼야 하는지" 텍스트·도식·색상으로 교정 안내 → 정답 자세를 일정 시간 유지하면 통과.

### 1.2 핵심 사용자 시나리오 (이 흐름이 끝까지 동작하면 성공)
1. 사용자가 상단에서 오카리나 종류 **"12홀 횡형"** 을 고른다(기본값).
2. 사용자가 **"도(C5)"** 버튼을 누른다.
3. 화면 우측에 "도(C5)"의 표준 운지 도식(앞면 10구멍 + 뒷면 2구멍 + **입김 세기 표시**)이 뜬다.
4. 사용자가 웹캠 앞에서 양손을 카메라 쪽으로 들고 오카리나 운지 모양을 만든다(에어 모드).
5. 앱이 실시간으로 각 손가락의 **막음/엶(0/1)** 을 인식하고, 도식 위에 **현재 내 손 상태**를 겹쳐 보여준다.
6. 틀린 구멍은 빨강, 맞은 구멍은 초록으로 표시되고, 하단에 `"오른손 약지로 구멍을 막으세요"` 또는 `"왼손 약지를 떼세요(교차운지)"` 같은 교정 문장이 뜬다.
7. 모든 감지 요소가 맞은 상태를 1.5초 유지하면 "정답! ✅" → 다음 계이름으로 진행 옵션 제공.

### 1.3 MVP 범위(꼭 만들 것) vs 이후(Phase 6+)
- **MVP**: **12홀 횡형**의 **앞면 8개 주요 구멍**(양손 검지·중지·약지·새끼 = `L2 L3 L4 L5 / R2 R3 R4 R5`)을 손가락 막음/엶으로 판정 + 교정.
  대상 음역은 자료에 있는 **A4(낮은라)~E6(높은미) 중 8구멍만으로 유일하게 구분되는 음**(5.6의 `MVP_NOTES`).
- **이후**:
  - 뒷면 **엄지구멍(L1·R1)** 자동 감지(가림 → 수동 폴백 우선),
  - **보조구멍(LS·RS)·반구멍(half-hole)·고급 교차운지** 정밀 판정,
  - **다른 종류**(4홀·6홀·10홀·더블/트리플) 운지 데이터 추가,
  - 입김(음역) 안내 강화, 실연주(가로 파지) 모드, 점수/연습 기록, 3D 고스트 핸드 오버레이.

## 2. 제공된 자료(`01 Source/`)와 활용 지침

`01 Source/` 폴더에 다음이 있다. **운지 데이터의 정답(ground truth)은 이 자료들, 특히 12홀 음별 SVG다.**

| 파일/폴더 | 용도 |
|---|---|
| `ocarina_fingering_charts_by_type(Claude)[All]/음별_20/*.svg` | **★ 최우선 근거(12홀 20음)**. 음 하나당 SVG 1장: 음이름·계이름·MIDI, **입김 세기(5칸)**, **앞면 10구멍 + 뒷면 2구멍**의 막음/엶이 원의 `fill` 색으로 인코딩됨. 파일명 = 계이름 id(`do.svg`, `reS.svg`, `lowLa.svg`, `highMi.svg` …). **이 색상을 파싱해 운지 JSON을 만든다.** |
| `…[All]/오카리나_온음_운지표_포스터.svg` | 12홀 자연음 전체를 한 장에 모은 포스터(사람 눈 대조용) |
| `…[All]/A4페이지_6쪽/page_*.svg` | 인쇄용 6쪽 레이아웃(동일 데이터) |
| `ocarina_fingering_charts_by_type(Chat_GPT).html` / `.svg` / `.pdf` | **★ 종류별 개요**. 4홀·6홀·10홀·12홀·더블/트리플 5종의 대표 운지표 + 범례. "검은 원=막기, 흰 원=열기, 반원=반구멍/분할구멍". **다른 종류 데이터의 1차 근거**(부록 C) |
| `ocarina_fingering_charts_by_type(Claude).html` / `.svg` / `.pdf` | 12홀 운지표(클로드판). 도식 스타일·색상 톤 참조 |

**활용 지침(AI에게):**
- 12홀 운지 데이터는 `음별_20/*.svg`의 원(circle) `fill`을 파싱해 **단일 출처(single source of truth)** 로 추출한다. 색 규칙:
  - **채워짐(`#E0407B` 분홍 = 왼손 / `#37474F` 진회색 = 오른손) = 막음(닫힘, 1)**
  - **흰색(`#FFFFFF`, 회색 테두리) = 엶(열림, 0)**
  - (참고) 다른 종류 차트의 **반원/분할** 표기 = **반구멍(0.5)** — MVP 자동 채점에서 제외, 도식에만 표시.
- 입김(`입김` 라벨 아래 5칸 사각형) 중 `fill="#2E5AAC"`(파랑) 칸 수 = **입김 세기 1~5**. 음역(옥타브)을 가르는 핵심이지만 **카메라로 감지 불가** → 운지 판정에서 제외하고 **도식에 안내 정보로만** 표시(3.5).
- 운지 도식 UI는 위 SVG 스타일(오카리나 몸통 실루엣 + 앞면/뒷면 분리, ● 막음 / ○ 엶, 왼손=분홍·오른손=진회색)을 **그대로 차용**해 기존 자료와 시각적 일관성을 줄 것. 가능하면 원본 SVG의 **구멍 좌표를 그대로 재사용**(부록 B의 좌표표).
- 데이터를 **종류별로 분리**해 설계한다: `ocarinaData[typeId].notes[noteId]`. 12홀을 완성형으로 만들고, 나머지 종류는 같은 스키마로 확장(부록 C).
- ⚠️ **데이터를 임의로 지어내지 말 것.** 추출이 모호한 음(반구멍·고급 교차운지·뒷면 엄지)은 `confidence`를 낮추고 `01 Source` 차트와 눈으로 대조해 확정한다. 확정 못 하면 멈추고 질문.

## 3. 핵심 기술 결정 & 근거 (반드시 이 전제로 구현)

> 이 절은 "왜 이렇게 만드는가"이며, 구현의 토대다. 마음대로 바꾸지 말 것.

### 3.1 캡처 자세 = "연습(에어) 모드"를 기본으로
- **문제**: 실제 오카리나 연주 자세는 악기를 양손으로 감싸 쥐고 **앞면 구멍이 카메라 반대쪽(연주자 정면 바깥)** 을 향한다. 손가락이 악기 뒤로 가려져(occlusion) 단안 웹캠으로 막음/엶 구분이 매우 불안정하다. 뒷면 엄지구멍은 아예 손바닥에 완전히 가려진다.
- **결정**: MVP의 기본 캡처 자세는 **"양손을 카메라 쪽으로 들고, 손가락 끝이 보이게 하여 오카리나 운지 모양을 따라 하는 연습(에어) 모드"**.
  - 손바닥은 비스듬히 아래/앞, 손가락 끝이 카메라에 보이도록. **왼손은 화면 왼쪽(악기 머리 쪽), 오른손은 화면 오른쪽(악기 꼬리 쪽).**
  - "구멍을 막음" = 손가락을 **구부려 끝을 아래로 내림**, "엶" = 손가락을 **펴서 들어 올림** 으로 흉내 낸다. 이렇게 하면 각 손가락의 막음/엶이 명확히 보여 인식 안정성이 크게 올라간다.
- 실연주 파지(악기를 쥔 가로 자세) 인식은 **Phase 7 이후 도전 과제**로 분리(상단/측면 보조 카메라, 악기 평면 검출 등 → 10장·부록 D 고도화 전략).

### 3.2 손 인식 엔진 = MediaPipe Tasks Vision · HandLandmarker
- 브라우저에서 무료·클라이언트 사이드로 **양손 각각 21개 3D 랜드마크 + 좌/우(handedness)** 를 제공.
- 학습 불필요(사전학습 모델). `runningMode: "VIDEO"`, `numHands: 2`.
- CDN로 로드(버전은 최신 안정판으로, 설치 시 공식 문서로 확인):
  - 모듈: `@mediapipe/tasks-vision`
  - 모델: `hand_landmarker.task` (Google 호스팅 .task 파일 URL).

### 3.3 구멍 막음/엶 판정 = "캘리브레이션 기준 + 상대 특징"
- **문제**: "구멍을 막았다/뗐다"를 절대 좌표 임계값으로 보면 사람·거리·각도마다 달라 불안정.
- **결정**: 사용 시작 시 **두 자세를 캘리브레이션**(8손가락 전부 막음 / 전부 엶)으로 받아 **손가락별 임계값을 개인화**한다. 특징값은 손 크기로 정규화한 상대값을 쓴다(6.2, 6.4).
- 오카리나는 색소폰의 옥타브 키처럼 별도 토글 요소가 없어, **MVP 캘리브레이션은 2단계(막음/엶)** 로 색소폰보다 단순하다.

### 3.4 거울(미러링) 처리
- 셀피 캠은 좌우 반전되어 보인다. **영상은 미러로 보여주되(자연스러움)**, 판정은 MediaPipe가 주는 `handedness`(실제 좌/우)를 기준으로 한다. 표시용 좌표와 논리용 좌/우를 혼동하지 말 것.
- 오카리나 매핑: 연주자 기준 **왼손 = 악기 머리(취구) 쪽 구멍(L2~L5, 뒷면 L1), 오른손 = 꼬리 쪽 구멍(R2~R5, 뒷면 R1)**. 미러 영상에서 화면상 좌/우와 실제 손이 바뀌어 보이므로 반드시 `handedness`로 매핑.

### 3.5 옥타브 키가 없다 = 음역은 "입김"이 가른다 (오카리나의 핵심 차이)
- 색소폰은 옥타브 키, 플루트는 입술/배음으로 음역을 가르지만, **오카리나는 입김(호흡 압력)** 으로 같은 운지의 음정을 끌어올린다. 자료의 `입김` 5칸 표시가 그 세기다(낮은음=여리게 1칸, 높은음=세게 5칸).
- **입김은 카메라(손 추적)로 감지 불가** → **운지 판정 대상이 아님**. 다만:
  - 도식에 **입김 세기 게이지(🌬️ 1~5)** 를 정보로 표시하고, 정답 시 `"입김을 조금 더 세게/여리게"` 같은 **코칭 힌트**(텍스트/선택적 음성)로만 안내한다.
  - 같은 손가락 패턴이라도 입김으로 음이 갈리는 경우가 있으므로(8요소 MVP의 모호성), 그런 음은 5.6의 `ambiguousWith`로 묶어 UI에 명시한다.

### 3.6 MVP에서 검출하는 요소 = 앞면 8구멍(8손가락) = 8요소
- LH 검지/중지/약지/새끼(`L2 L3 L4 L5`), RH 검지/중지/약지/새끼(`R2 R3 R4 R5`) = 음을 결정하는 주요 8구멍(앞면).
- **MVP 제외(가림·미세·연속값 → Phase 6+)**:
  - **뒷면 엄지구멍(L1·R1)**: 손바닥 뒤라 카메라로 거의 안 보임. 대부분의 음에서 닫혀 있고 **최고음(예: highRe·highReS·highMi)에서만 열림** → MVP는 **기본 "닫힘" 가정 + 수동 토글 폴백**(6.5).
  - **보조구멍(LS·RS)**: 작은 분할구멍. 새끼 동작과 함께 움직여 독립 추적 어려움 → MVP는 **인접 새끼와 동일 상태로 가정**하거나 도식에만 표시.
  - **반구멍(half-hole)**: 손끝을 절반만 덮는 연속 동작. 단안 웹캠으로 정량화 불가 → MVP 제외, 도식·안내에만 노출.
- ⚠️ **한계 명시(반드시 UI에 표기)**:
  - 8구멍만으로는 **뒷면 엄지/보조구멍/입김으로만 갈리는 음**(특히 최고음역, 일부 반음)이 서로 같은 손모양으로 보여 구분 불가 → `ambiguousWith`로 묶어 안내.
  - 자동 판정이 불안정한 요소는 **수동 폴백**(키보드/클릭 토글)으로 우아하게 degrade.

### 5.2 구멍 식별자 & 손가락 매핑 (12홀 — `음별_20` SVG에서 검증 추출)
손가락 번호 규칙: **1=엄지, 2=검지, 3=중지, 4=약지, 5=새끼** (양손). 뒷면 = 엄지. `S`=보조구멍.

| 구멍 ID | 손가락 | 위치 | MVP 감지 |
|---|---|---|:--:|
| `L1` | 왼손 엄지 | 뒷면 | ✗ (가림→수동) |
| `L2` | 왼손 검지 | 앞면 | ★ |
| `L3` | 왼손 중지 | 앞면 | ★ |
| `L4` | 왼손 약지 | 앞면 | ★ |
| `L5` | 왼손 새끼 | 앞면 | ★ |
| `LS` | (왼손 보조) | 앞면 작은 구멍 | ✗ (새끼와 동반) |
| `R1` | 오른손 엄지 | 뒷면 | ✗ (가림→수동) |
| `R2` | 오른손 검지 | 앞면 | ★ |
| `R3` | 오른손 중지 | 앞면 | ★ |
| `R4` | 오른손 약지 | 앞면 | ★ |
| `R5` | 오른손 새끼 | 앞면 | ★ |
| `RS` | (오른손 보조) | 앞면 작은 구멍 | ✗ (새끼와 동반) |

값: `1 = 막음(닫힘 ●)`, `0 = 엶(열림 ○)`, `0.5 = 반구멍`(다른 종류에서만, MVP 제외).
**MVP 감지 8요소** `DETECTABLE = ["L2","L3","L4","L5","R2","R3","R4","R5"]`.

> 구멍의 SVG 좌표(앞면/뒷면)는 **부록 B 표**에 그대로 있다. 도식은 이 좌표를 재사용해 원본과 1:1로 맞춘다.

### 5.3 계이름 매핑
오카리나 운지표는 **C조 기준**. `도=C, 레=D, 미=E, 파=F, 솔=G, 라=A, 시=B`(반음 `#`/`b`). 음 `id`에 옥타브를 포함.
파일명 ↔ 음 id 예: `lowLa`=A4, `lowLaS`=A#4, `lowSi`=B4, `do`=C5, `doS`=C#5, `re`=D5, `reS`=D#5, `mi`=E5, `fa`=F5, `faS`=F#5, `sol`=G5, `solS`=G#5, `la`=A5, `laS`=A#5, `si`=B5, `highDo`=C6, `highDoS`=C#6, `highRe`=D6, `highReS`=D#6, `highMi`=E6. (총 20음, A4~E6 반음 포함)

### 5.4 운지 데이터 추출 규칙 (음별 SVG → JSON)
각 `음별_20/*.svg`에서:
1. 상단 텍스트에서 **계이름·음이름·MIDI** 추출.
2. **입김**: `transform="translate(300,58)"` 그룹의 사각형 5개 중 `fill="#2E5AAC"` 개수 = `breath`(1~5).
3. **앞면**(`앞면` 그룹의 내부 SVG, viewBox `0 0 900 470`): 원 10개를 **고정 좌표**로 식별(부록 B) → `fill`이 흰색이면 0(엶), 분홍/진회색이면 1(막음).
4. **뒷면**(`뒷면` 그룹): 원 2개(엄지) 동일 규칙. 해치 패턴 타원은 **취구(windway)** 이므로 구멍 아님 — 무시.
5. 하단 한 줄 설명 텍스트는 `tip`(교정/안내 문구 시드)로 저장.
6. 제목 옆 배지(`교차운지`/`고급`)가 있으면 `flags`에 기록.

## 6. 손가락 막음/엶 검출 알고리즘 (구현 핵심)

### 6.1 MediaPipe 랜드마크 인덱스(손당 21개)
```
손목 0
엄지: CMC1  MCP2  IP3   TIP4
검지: MCP5  PIP6  DIP7  TIP8
중지: MCP9  PIP10 DIP11 TIP12
약지: MCP13 PIP14 DIP15 TIP16
새끼: MCP17 PIP18 DIP19 TIP20
```
손가락 ↔ 랜드마크: 검지=(5,6,7,8) 중지=(9,10,11,12) 약지=(13,14,15,16) 새끼=(17,18,19,20) 엄지=(1,2,3,4).

### 6.2 손가락별 특징값(정규화) — 막을수록 작아지는 1D 값
검지/중지/약지/새끼(`L2~L5`, `R2~R5`) 각각에 대해 두 특징을 계산하고 가중 합한다:
1) **굽힘각(curl)**: `a = PIP−MCP`, `b = TIP−PIP`, `θ = angle(a,b)`. 펴서 들면(엶) θ↓, 구부려 막으면 θ↑ — 부호 방향은 캘리브레이션으로 결정.
2) **끝높이(tip lift)**: 손바닥 평면(MCP들을 잇는 선)에 대한 TIP의 수직 거리를 손 크기(`|MCP_index − MCP_pinky|`)로 정규화. 엶이면 끝이 들려 값↑.
```js
function fingerFeature(lm, MCP, PIP, TIP, handScale){
  const curl    = angleBetween(sub(lm[PIP],lm[MCP]), sub(lm[TIP],lm[PIP]));
  const tipLift = perpDistanceToMcpLine(lm[TIP], lm) / handScale;
  return { curl, tipLift };
}
```
> 한 특징만으로 불안정하면 둘을 정규화 후 평균. 어느 특징이 잘 먹는지는 캘리브레이션 데이터로 자동 선택해도 좋다. **오카리나 새끼손가락(L5/R5)** 은 굽힘이 작고 떨림이 커서 가장 어려움 → 스무딩 윈도우를 더 크게.

### 6.3 뒷면 엄지구멍(L1·R1) — MVP는 수동 폴백
- 에어 모드에서도 엄지는 손바닥 뒤로 가기 쉬워 자동 신뢰도가 낮다. **MVP 기본값 = "닫힘(1)" 가정** + **수동 토글**(키보드 `Z`=왼엄지, `X`=오른엄지 또는 화면 버튼)로 최고음(엄지 엶) 대응. 자동 검출은 Phase 6에서 엄지 굽힘각으로 시도.

### 6.4 캘리브레이션(개인화 임계값)
1. UI: "① 여덟 손가락 모두 **막은**(구부린) 모양" 2초 수집 → 손가락별 특징 평균 = `downRef`.
2. "② 여덟 손가락 모두 **편**(엶) 모양" 2초 수집 → `upRef`.
3. 손가락별 임계값 `thr = (downRef+upRef)/2`, 방향(`downIsHigher`) 저장.
4. `localStorage`에 저장해 다음 방문 시 재사용(재캘리브레이션 버튼 제공).
```js
isClosed(f, cal) => cal.downIsHigher ? (f > cal.thr) : (f < cal.thr)  // true=막음(1)
```

### 6.5 스무딩 & 디바운스(떨림 방지 — 필수)
- 요소별 상태에 **EMA 또는 최근 N=5프레임 다수결** 적용. 새끼는 N=7 권장.
- 상태 전환은 **히스테리시스**(막음↔엶 임계에 ±마진)로 깜빡임 방지.
- 손이 화면에 없거나 신뢰도 낮은 프레임은 **"판정 보류"(회색)** — 빨강 오판 금지.
- **양손 필수**: 오카리나는 양손을 모두 본다. 한 손만 검출되면 "양손을 모두 보여주세요" 안내.

### 6.6 출력
```js
currentState = {
  L2:1,L3:1,L4:1,L5:1, R2:1,R3:1,R4:0,R5:0,   // MVP 8요소
  L1:1,R1:1, thumbSource:"manual",            // 엄지(수동/자동)
  confidence:0.0~1.0, handsVisible:2
}
```

## 9. 단계별 구현 로드맵 (각 Phase = 1 작업 단위)

> **AI에게**: 한 Phase씩 구현하고, 끝나면 "지금 브라우저에서 무엇이 보이는지" + 완료 조건 충족 여부를 보고한 뒤 다음 Phase로.

### PHASE 0 — 프로젝트 스캐폴딩

## 10. 정확도·한계·튜닝 가이드 (AI가 사용자에게 반드시 안내)

- **단안 웹캠 한계**: 손가락 막음/엶은 작은 움직임이라 100% 정확하지 않음 → 캘리브레이션·조명·손 전체가 화면에 들어오는 것이 중요.
- **오카리나 고유 한계**:
  - **입김(음역)은 카메라로 감지 불가** → 같은 손모양이 입김으로 다른 음이 될 수 있음(모호성 UI 표기).
  - **뒷면 엄지구멍·보조구멍은 가림** → MVP 수동/가정 처리.
  - **반구멍**은 연속값이라 MVP 자동 채점 제외.
  - **종류마다 운지 체계가 다름**(Asian/Italian 고음 운지, 제조사 차이) → 자료는 C조 대표 운지이며 실제 악기는 튜너로 확인 권장(자료 `정확성 기준` 문구 그대로 안내).
- **튜닝 포인트**: 특징 가중치(curl vs tipLift), 임계 마진(히스테리시스), 스무딩 윈도우 N(새끼는 크게), 정답 유지 시간.
- **권장 환경**: 밝은 정면 조명, 양손이 화면 60% 이상 차지, 단색 배경, 손가락 끝이 카메라를 향하게.

### 부록 A. 참고 코드 스니펫 (MediaPipe 초기화 — 버전은 공식 문서로 확인)
```js
// handTracking.js
import { HandLandmarker, FilesetResolver } from
  "https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision/vision_bundle.mjs";

export async function createHandLandmarker(){
  const vision = await FilesetResolver.forVisionTasks(
    "https://cdn.jsdelivr.net/npm/@mediapipe/tasks-vision/wasm");
  return await HandLandmarker.createFromOptions(vision, {
    baseOptions: { modelAssetPath:
      "https://storage.googleapis.com/mediapipe-models/hand_landmarker/hand_landmarker/float16/1/hand_landmarker.task" },
    runningMode: "VIDEO",
    numHands: 2,
  });
}
// 루프: const res = landmarker.detectForVideo(video, performance.now());
// res.landmarks[i] = 21 점, res.handednesses[i][0].categoryName = "Left"/"Right"
```

### 부록 C. 종류별 데이터 확장 가이드 (Chat_GPT 차트 기준)
`ocarina_fingering_charts_by_type(Chat_GPT).html`의 종류별 페이지에서 같은 스키마로 추출:
- **4홀 펜던트**(앞면 4): 손가락 적고 **반구멍으로 반음**을 만듦 → 반구멍 표기 필수(MVP 자동 채점 제외).
- **6홀 펜던트**(앞면 4 + 엄지 2): 4홀보다 D·E 안정. 반구멍 多.
- **10홀 횡형**(앞면 8 + 엄지 2): 12홀에서 보조구멍 2개 뺀 형태 → `L2~L5/R2~R5 + L1/R1`. **12홀 로직 거의 재사용**.
- **12홀 횡형**: 본 문서 기준(완성형).
- **더블/트리플**: 2~3 챔버, 1챔버 후 오른손으로 2·3챔버 이어 연주 → 후순위.
- 공통 범례: **검은 원=막기, 흰 원=열기, 반원=반구멍/분할 일부 열기**. C조 기준이며 제조사·고음 체계(Asian/Italian)·조율에 따라 일부 반음 운지가 다를 수 있음(UI에 그대로 고지).
